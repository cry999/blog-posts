# ABC 441 E: A > B substring

[問題](https://atcoder.jp/contests/abc441/tasks/abc441_e)

## 問題概要

`A`, `B`, `C` の3種類の文字からなる文字列 S の部分文字列で `A` が `B` よりも多くなる部分文字列の個数を数える。

## つまづいたポイント

そのまま `A` と `B` の文字数の累積和を数え上げて比較しようとした。
しかし、そうすると `A` の文字数と `B` の文字数を別々に管理するため、効率的な比較ができなかった。

## 面白いと感じたポイント

### 「`A` が `B` よりも多い部分文字列」←→「`A-B` が正となる区間 `[l, r]`」の言い換え

解説にもあるとおり、「A が B よりも多い i 文字目から j 文字目までの部分文字列」は、先頭から i 文字目までの A, B それぞれの出現回数を `A[i]`, `B[i]` と表現すれば `A[j] - A[i-1] > B[j] - B[i-1]` と表現できる。
これを変形することで、 `A[j] - B[j] > A[i-1] - B[i-1]` と表現できる。つまり、A と B の出現回数の差としてまとめて扱えるようになる。

### `A-B` が正になる区間を累積値を利用して数え上げる方法

上述の変形により、 $D_i := i 文字目までに出てくる A と B の出現回数の差$ と定義すると $0 \leq j < i \leq N$ かつ $D_i > D_j$ となる $(j, i)$ の組を数え上げることになる。

ただ、例えば、i を動かしながら、条件を満たす j をいちいち数えていたら N に関するループを二重に回すことになるので $O(N^2)$ かかることになる。

そこで今回の工夫がある。

注目するのが $D_i$ から $D_{i+1}$ への変化は上下高々 1 しかないということ。
これによって、以下のような対応を考えると現在の $D_i$ 未満の $D_j$ ($j < i$) の個数を累積することができる。
以下の説明ではこの累積した値を $C$ とする

#### $S_i$ が A の時

$C$ には $D_i$ 未満となる $D_j (j < i)$ の個数が保存されている。
$S_i = 'A'$ のため、$D_{i+1} = D_i + 1$ となり、次は $D_{i+1}$ 未満の個数が欲しい。
ここで、$D_{i+1} = D_i + 1$ のため、$D_{i+1}$ 未満は $D_i$ 以下であり、$C$ には $D_i$ 未満の個数がすでにあることと合わせて $C + (D_i の個数)$ が得られれば目的は達成される。

#### $S_i$ が B の時

$C$ には $D_i$ 未満となる $D_j (j < i)$ の個数が保存されている。
$S_i = 'B'$ のため、$D_{i+1} = D_i - 1$ となり、次は $D_{i+1}$ 未満の個数が欲しい。
ここで、$D_{i+1} = D_i - 1$ のため、$D_{i+1}$ 未満は $D_i - 2$ 以下であり、$C$ には $D_i$ 未満の個数がすでにあることと合わせて $C - (D_i - 1 の個数)$ が得られれば目的は達成される。

#### $S_i$ が A, B 以外の時

$C$ には $D_i$ 未満となる $D_j (j < i)$ の個数が保存されている。
$S_i$ は A でも B でもないため、$D_{i+1} = D_i$ となり、次も $D_i$ 未満の個数で良い。
そのため、そのまま $C$ を利用可能である。


このようにして、$C$ を用いることで全体で $O(N)$ の計算量で目的を達成できる。

## 解答例

ほとんど解説と同じだが python による解答例

```python
N = int(input())
S = input()


# diff: 現在までの「A の出現回数 - B の出現回数」
# [-N, N] の範囲の値を取るため、配列の添字として使えるように +N で補正する
diff = 0 + N

# counter[d] := 現在より過去の diff が d だった回数。
# つまり、「A の出現回数 - B の出現回数」が d-N (N は補正分) だった prefix の個数。
counter = [0] * (2 * N + 1)

# 空の prefix をカウントしておく。
counter[diff] += 1


# sum_less: 過去の diff のうち、現在の diff より小さいものの個数
# diff は +-1 ずつしか変化しない。
# A が出現すると diff+1 になるので、counter[diff] 分だけ増える
# B が出現すると diff-1 になるので、counter[diff-1] 分だけ減る
sum_less = 0
ans = 0

for s in S:
    if s == "A":
        # diff+1 になるので、diff 以下の個数を答えに足し合わせたい。
        sum_less += counter[diff]
        diff += 1
    elif s == "B":
        # B が出現すると diff は 1 減る。
        # 新しい diff (diff-1) 未満の個数にするため、新しい diff と等しい過去の個数を引く。
        diff -= 1
        sum_less -= counter[diff]

    # 答えに現在の diff より小さい diff の個数を足し合わせておく。
    ans += sum_less

    # 次以降のループように現在の diff をカウントしておく
    counter[diff] += 1

print(ans)
```
