# アウトラインテンプレートの作成・改善

テーマ（技術解説・競プロ系 / ノウハウ系 / アイデア系 など）ごとの再利用可能なアウトライン雛形を管理する。
テンプレートは `.claude/article-templates/<slug>.md` に **1 テーマ 1 ファイル** で保存する。`create-outline` スキルはこのディレクトリのテンプレートを使って実際の記事アウトラインを組み立てる。

このスキルがやることは「雛形そのもの」の整備であり、特定の記事のアウトライン作成ではない。個別記事のアウトラインが欲しい場合は `create-outline` を使う。

## テンプレートの保存場所と形式

- 場所: `.claude/article-templates/<slug>.md`（slug は英小文字＋ハイフン。例: `tech-competitive`）
- 形式: frontmatter ＋ 見出し構成。各見出しに「そのセクションに何を書くか」を箇条書きで添える。本文は書かない。

frontmatter:

```yaml
---
theme: 技術解説・競プロ系      # 人が読むテーマ名
slug: tech-competitive         # ファイル名と一致させる
aliases:                       # このテーマを指す呼び方（マッチ判定に使う）
  - 競プロ
  - atcoder
description: いつ使うテンプレートかの 1 行説明
---
```

## 工程

### 1. 何をしたいかを判定する

ユーザーの依頼が「新規作成」「改善」「一覧確認」のどれかを見極める。まず `.claude/article-templates/` の既存ファイルを確認してから動く。

### 2-A. 新規テンプレートを作成する場合

1. **テーマをヒアリング**: どんな系統の記事向けか（例: 「障害対応の振り返り（ポストモーテム）系」）。対象読者と典型的な記事の流れも聞く。
2. **重複チェック**: 既存テンプレートと役割が被らないか確認する。近いものがあれば「改善」に切り替える。
3. **見出し構成を設計**: そのテーマで毎回使える 4〜6 個程度のセクションを組む。各セクションに何を書くかを箇条書きで添える。Zenn 記事の規約（CLAUDE.md 参照、全文日本語）に沿う。
4. **ファイルを作成**: `.claude/article-templates/<slug>.md` に frontmatter 付きで書き出す。slug は英小文字＋ハイフン、`aliases` にユーザーが使いそうな呼び方を入れる。
5. ユーザーに提示して合意を取る。

### 2-B. 既存テンプレートを改善する場合

1. **対象を特定**: どのテンプレートか（theme / slug / alias で照合）。曖昧なら候補を提示して選んでもらう。
2. **現状を読む**: 対象ファイルを読み、何を改善したいか（セクションの過不足、説明の薄さ、順序、新しい観点の追加など）をヒアリングする。
3. **差分を提案**: 変更点を要約してユーザーに見せ、合意を取る。雛形の汎用性を保つこと（特定記事に寄せすぎない）。
4. **ファイルを更新**: 合意した内容で上書きする。frontmatter の `description`/`aliases` も必要なら更新する。

### 2-C. 一覧を確認する場合

`.claude/article-templates/` の各ファイルの frontmatter（theme / description）を読み、テーマ一覧を表形式などで提示する。

## 注意

- テンプレートは「どの記事にも使い回せる骨格」。特定の問題・特定のライブラリに依存した中身を書き込まない。プレースホルダ（「要点①: 〜」など）で汎用性を残す。
- 本文は書かない。見出しと各セクションの要点メモにとどめる。
- 既存テンプレートを上書きする前に必ず現状を読む。意図せぬ削除をしない。slug を変える場合は旧ファイルの扱い（リネーム）をユーザーに確認する。
- 全文日本語で書く。
- テンプレートを増やしたら、`create-outline` がそれを使えること（このディレクトリを参照する旨）を前提に整合性を保つ。
