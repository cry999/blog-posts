## TL;DR

- ABC160 D は制約 $N \le 2000$ なので **想定は $O(N^2)$**（全ペアの距離を求める。各頂点から BFS するなど）。それで十分間に合う。
- でもこのグラフは「**直線のしっぽ → サイクル → 直線のしっぽ**」というきれいな 3 部構造を持つ。これを使うと、ペアを **4 カテゴリ**（直線内 / サイクル内 / しっぽ↔サイクル / しっぽ↔しっぽ）に分け、各カテゴリの「距離 $d$ の組数」を **閉じた式や差分配列（imos 法）** で数えられる。
- 結果、距離ごとのヒストグラムを **$O(N)$** で構築できる。「**1 ペアずつ距離を測る**」より「**グラフの形を読む**」のが効く、という話。

## この問題について

- ABC160 D（パスグラフに辺を 1 本足したグラフで、各 $k$ について最短距離が $k$ のペア数を数える）
- 問題ページ: https://atcoder.jp/contests/abc160/tasks/abc160_d
- この記事で伝えたいこと: 想定の $O(N^2)$ で通る問題を、**グラフの構造分解**で $O(N)$ まで落とした話。距離を 1 ペアずつ測るのではなく、**構造ごとにまとめて数える**発想がキモ。

## 問題概要

- $1$ から $N$ までの頂点を持つ無向グラフ $G$。辺は次の $N$ 本:
  - $i = 1, \dots, N-1$ について頂点 $i$ と $i+1$（＝**1 本の直線パス**）
  - 加えて頂点 $X$ と頂点 $Y$ の間に 1 本（**ショートカット辺**）
- $k = 1, 2, \dots, N-1$ について、$1 \le i < j \le N$ で頂点 $i, j$ の最短距離がちょうど $k$ である組 $(i, j)$ の個数を出力する。

![問題のグラフ例（N=7, X=2, Y=5）。ショートカット辺で頂点 1→6 の最短距離が 5 から 3 に縮む様子](/images/20260622-atcoder-abc160-d-cycle-decomposition/problem-overview.png)

- 制約: $3 \le N \le 2 \times 10^3$、$1 \le X, Y \le N$、$X + 1 < Y$（＝サイクルの長さは 3 以上）。
- 入出力例（公式サンプル 1）: 入力 `5 2 4`（$N=5,\ X=2,\ Y=4$）→ 出力は $k=1,2,3,4$ の順に `5`, `4`, `1`, `0`。距離 1 のペアは $(1,2),(2,3),(3,4),(4,5)$ に加え、ショートカット辺で繋がる $(2,4)$ も含めて 5 組、という具合に「距離ごとの組数」を答える。

## 考察

### 要点①: もしこれが「単なる直線」なら簡単 — 距離 $d$ の組は $n - d$ 個

ショートカット辺がなければ、グラフはただの直線パス $1 - 2 - \dots - N$。このとき距離 $d$ のペアは $(i, i+d)$ の形に限られ、$i$ は $1$ から $n - d$ まで動けるので、**距離 $d$ のペアはちょうど $n - d$ 個**。$d = 1$ なら隣接の $n - 1$ 組、$d = n - 1$ なら両端の $1$ 組、と素直に減っていく。

![直線で距離 d のペア (i, i+d) を数える図（n=6, d=2 では (1,3),(2,4),(3,5),(4,6) の 4 本 ＝ n−d）](/images/20260622-atcoder-abc160-d-cycle-decomposition/line-count.png)

各ペアの距離をいちいち測らなくても、「$(i, i+d)$ という見方をするだけ」で一発で本数が出るのが直線のうれしさだ。問題を難しくしているのは**ショートカット辺ただ 1 本**で、これがサイクルを作り「直線をまたぐ近道」を生む。この近道の扱いが要点②以降の主題になる。

なお想定解は、この近道を素直に織り込んで全ペアの距離を求める $O(N^2)$。$N \le 2000$ なら十分間に合うが、ここではグラフの形を活かして $O(N)$ まで落とす。

### 要点②: グラフを「しっぽ＋サイクル＋しっぽ」に分解する

- このグラフはショートカット辺によって、ちょうど次の 3 部構造になる:
  - **左しっぽ**: 頂点 $1, \dots, X-1$（サイクルにぶら下がる直線、長さ `left_tail = X - 1`）
  - **サイクル**: 頂点 $X, X+1, \dots, Y$（直線部分 ＋ ショートカット辺で閉じた閉路、長さ `cycle = Y - X + 1`）
  - **右しっぽ**: 頂点 $Y+1, \dots, N$（長さ `right_tail = N - Y`）

![グラフを左しっぽ・サイクル・右しっぽの 3 部に分解した図](/images/20260622-atcoder-abc160-d-cycle-decomposition/decomposition.png)

- 頂点対 $(i, j)$ を、両端がどの部分に属するかで **4 カテゴリ** に分ける:
  1. **直線内**（しっぽ内 ＋ 端のサイクル接続点まで）: 普通の直線パス上の距離
  2. **サイクル内**: 両端ともサイクル上 → 閉路上の距離（時計回り / 反時計回りの短い方）
  3. **しっぽ↔サイクル**: 片方がしっぽ、もう片方がサイクル上
  4. **しっぽ↔しっぽ**: 左しっぽと右しっぽ（必ずサイクルを跨ぐ）
- この 4 カテゴリは**互いに重ならず、全ペアを尽くす**。気をつけるのはサイクルの接続点 $X$ / $Y$ をどちらに入れるかだけで、本実装は「接続点を直線側に含める」と決めている（後述の `add_line(tail+1)`）。各カテゴリで距離 $d$ の組数を数え、全部足せば答えのヒストグラムになる。

### 要点③: 4 カテゴリそれぞれを、閉じた式 or 差分配列（imos）で数える

要点②で分けた 4 カテゴリは、どれも距離 $d$ に対する組数が**きれいな形**になる。直線内・サイクル内・しっぽ↔しっぽは**閉じた式**で、しっぽ↔サイクルは**差分配列（区分線形 → 範囲加算）**で数える。1 カテゴリずつ「数え方 → 図 → 対応するコード」の順に見ていく。

#### 直線内（`add_line`）

長さ $n$ の直線では、距離 $d$ の組は $(i, i+d)$ の形で $n - d$ 個（要点①の図そのもの）。閉じた式で一発。

```python
def add_line(n: int):
    for d in range(1, n):    # 距離 d = 1 .. n-1
        ans[d] += n - d      # 各距離ちょうど n - d 個
```

#### サイクル内（`add_cycle`）

サイクル（頂点 $X, X+1, \dots, Y$ の $n = Y - X + 1$ 個）でも、**直線とまったく同じく $(i, i+d)$ の組で数える**。違うのは、サイクルなので $i + d$ が $Y$ を超えたら先頭側へ**回り込む**ことだけ（$i + d \to i + d - n$、$n = Y - X + 1$）。

- まず **$d$ は $\lfloor n/2 \rfloor$ まで見れば十分**。サイクルでは「$i$ から時計回りに $d$ 歩」と「反時計回りに $n - d$ 歩」が同じ 2 頂点を指すので、ペア $(i, i+d)$ の本当の最短距離は $\min(d,\ n - d)$。つまり $d$ が $n/2$ を超えると、それは距離 $n - d\ (< n/2)$ のペアを**裏側から数え直しているだけ**で、より小さい $d$ で既に数え終えている。だから $d = 1, \dots, \lfloor n/2 \rfloor$ を見れば全距離を重複なく尽くせる（コードの `range(1, n // 2 + 1)` がこれ）。
- 各 $d$ について組数は: 直線では端があるので $i$ は $1, \dots, n - d$ の $n - d$ 個に限られた。サイクルは回り込める＝**端がない**ので、**$i$ は $n$ 頂点すべてを動ける** → 距離 $d$ の組は **$n$ 個**。直線の $n - d$ から「端の制約」が消えて $n$ になる、と見ると地続き。
- 例外は $d = n/2$（$n$ が偶数）のときだけ。このとき $(i, i+d)$ と $(i+d, i+2d) = (i+d, i)$ が**同じペアを指す**（ちょうど一周の半分なので表と裏が一致する）ので、2 回ずつ数えてしまい **$n/2$ 個**。

![サイクル内も (i, i+d) で数える図（n=6）。左 d=2(<n/2) は (i,i+2) が全 6 頂点ぶんで n=6 個、中 d=3(=n/2) は (i,i+3) の表裏が一致して n/2=3 個、右 d=4(>n/2) は同じペア {1,5} を逆回りなら n−d=2 歩で指す＝d=2 で既出なので数えない](/images/20260622-atcoder-abc160-d-cycle-decomposition/cycle-count.png)

```python
def add_cycle(n: int):
    for d in range(1, n // 2 + 1):   # 距離は 1 .. ⌊n/2⌋ まで
        if 2 * d == n:               # d = n/2（偶数長サイクルの対角）
            ans[d] += n // 2         # 対角だけ半分
        else:
            ans[d] += n              # それ以外はどの距離も n 個
```

`range(1, n // 2 + 1)` が「距離は最大 $\lfloor n/2 \rfloor$」、`if 2 * d == n` が「対角になるのは偶数長のときだけ」、その分岐の `n // 2` が上の図の右側（半分になる）に対応する。

#### しっぽ↔サイクル（`add_tail_and_cycle`）

ここは「区分線形 → 範囲加算」で計算する。しっぽの頂点 $p$（接続点 $E$ から $p$ 歩。$p = 1, \dots, \text{tail\_length}$）と、サイクル上の頂点 $c$ を結ぶペアを数える。$E$ を経由するしかないので、距離は

$$\mathrm{dist}(p, c) = p + (\text{$E$ から $c$ までのサイクル距離})$$

:::message
接続点 $E$ 自身との組は直線パート `add_line` で数え済みなので、ここでの相手は $E$ 以外のサイクル頂点。
:::

サイクル距離はサイクル内で見たとおり $1, 2, \dots, \lfloor n/2 \rfloor$（$n = \text{cycle\_length}$）まで動き、しかも**同じサイクル距離の頂点が左右両側に 2 つずつ**ある（対角だけ 1 つ）。だから 1 つの $p$ について距離の内訳はこうなる:

- 距離 $p+1,\ p+2,\ \dots,\ p + \lfloor (n-1)/2 \rfloor$ が **それぞれ 2 本**（サイクルの左右両側ぶん）
- $n$ が偶数なら、対角の距離 $p + n/2$ が **1 本**

![しっぽ頂点 p からサイクルへの距離の図（n=6, p=1）。左: サイクル頂点を E からの距離でラベルし同じ距離が両側 2 つ・対角 1 つ。右: p=1 の寄与が距離 2,3 に +2、対角の距離 4 に +1 という区間加算になる](/images/20260622-atcoder-abc160-d-cycle-decomposition/tail-cycle-count.png)

つまり 1 つの $p$ の寄与は「ある距離区間へ一律 $+2$、（偶数長なら）対角へ $+1$」という**区間加算**。これを距離 1 マスずつ足すと $p$ あたり $O(n)$ で全体 $O(\text{tail} \times n)$ になってしまう。そこで**差分配列（imos 法）で区間の両端だけ触り**、各 $p$ を $O(1)$ で済ませて、最後に一度だけ累積する。

```python
def add_tail_and_cycle(tail_length: int, cycle_length: int):
    diff = [0] * (N + 1)
    for p in range(1, tail_length + 1):
        # 距離 [p+1, p + ⌊(n-1)/2⌋] に一律 +2（左右両側ぶん）
        if p + 1 <= p + (cycle_length - 1) // 2:
            diff[p + 1] += 2
            diff[(cycle_length - 1) // 2 + p + 1] -= 2
        # 偶数長サイクルなら対角の距離 p + n/2 に +1
        if cycle_length % 2 == 0:
            diff[cycle_length // 2 + p] += 1
            diff[cycle_length // 2 + p + 1] -= 1
    cur = 0                         # 差分を累積して ans に反映（ここで一度だけ O(N)）
    for i in range(1, N):
        cur += diff[i]
        ans[i] += cur
```

- `diff[p+1] += 2` と `diff[(cycle_length-1)//2 + p + 1] -= 2`: 距離区間 $[p+1,\ p+\lfloor(n-1)/2\rfloor]$ への一律 $+2$ を、**両端 2 点を触るだけ**で表す（`(cycle_length-1)//2` が $\lfloor(n-1)/2\rfloor$）。これが図の右側の 2 本の高さ $+2$。
- `if cycle_length % 2 == 0`: 偶数長サイクルだけ対角がある。`diff[n//2 + p] += 1`（次の添字で $-1$）で対角距離 $p + n/2$ に 1 本だけ足す。これが図の右側の赤いバー $+1$。
- 最後の累積ループで差分を距離順に積み上げて `ans` に反映。`for p` 側は各 $p$ が $O(1)$ なので、全体は $O(\text{tail} + N) = O(N)$。

#### しっぽ↔しっぽ（`add_between_tails`）

左しっぽの頂点（接続点 $X$ から深さ $a$、$a = 1, \dots, \text{left\_tail}$）と、右しっぽの頂点（接続点 $Y$ から深さ $b$、$b = 1, \dots, \text{right\_tail}$）のペア。左右をまたぐにはサイクルを通るしかないが、$X$–$Y$ のショートカット 1 本を使う経路が最短なので、距離は

$$\mathrm{dist} = a + b + 1$$

その内訳（左しっぽを $a$ 歩 ＋ ショートカットを $1$ 歩 ＋ 右しっぽを $b$ 歩）は下図のとおり。なお、サイクルを大回りすると $a + (Y - X) + b$ になるが、$Y - X \ge 2 > 1$ なのでショートカットが必ず勝つ。

ここで $s = a + b$ とおくと距離は $s + 1$。距離 $s+1$ のペア数は「$a + b = s$ を満たす格子点 $(a, b)$ が、長方形 $1 \le a \le \text{left\_tail}$, $1 \le b \le \text{right\_tail}$ の中にいくつあるか」で数える。$a$ の動ける範囲は $b = s - a$ の制約から

$$\max(1,\ s - \text{right\_tail}) \le a \le \min(\text{left\_tail},\ s - 1)$$

で、その**区間の幅**が個数。

![しっぽ↔しっぽの図（左 3, 右 2）。左: 距離 a+1+b でショートカットが +1。右: 格子点 (a,b) で a+b=s の点の数が距離 s+1 のペア数になる（s=3 は 2 点で距離 4 が 2 個、合計 3×2=6 個）](/images/20260622-atcoder-abc160-d-cycle-decomposition/between-tails-count.png)

```python
def add_between_tails(left_tail: int, right_tail: int):
    for s in range(2, left_tail + right_tail + 1):   # s = a+b は 2 .. left+right
        lo = max(1, s - right_tail)        # b ≤ right_tail から a の下限
        hi = min(left_tail, s - 1)         # a ≤ left_tail, b ≥ 1 から a の上限
        ans[s + 1] += max(0, hi - lo + 1)  # 距離 s+1 に「区間の幅」ぶん
```

- `s` が $a + b$。距離はその $+1$ なので加算先は `ans[s + 1]`。$a, b \ge 1$ なので $s$ は $2$ から。
- `lo` / `hi` が上の $a$ の下限・上限、`max(0, hi - lo + 1)` がその区間に入る格子点の数（空なら 0）。図の各反対角線上の点の数に対応する。
- 合計すると $\sum_s (\text{点の数}) = \text{left\_tail} \times \text{right\_tail}$ で、左右しっぽの全ペアをちょうど一度ずつ数えている。しっぽが空（`left_tail` か `right_tail` が 0）なら各項が自然に 0 になり、特別扱いは要らない。

## 解法・実装

提出（Python）: https://atcoder.jp/contests/abc160/submissions/76881659

```python
N, X, Y = map(int, input().split())
ans = [0] * N

def add_line(n: int):
    # 長さ n の直線: 距離 d の組は n - d 個
    for d in range(1, n):
        ans[d] += n - d

def add_cycle(n: int):
    # 長さ n の閉路: 距離 d の組は n 個（偶数長の対角だけ n//2 個）
    for d in range(1, n // 2 + 1):
        if 2 * d == n:
            ans[d] += n // 2
        else:
            ans[d] += n

def add_tail_and_cycle(tail_length: int, cycle_length: int):
    # しっぽ↔サイクル: 各しっぽ頂点の寄与を差分配列(imos)で区間加算
    diff = [0] * (N + 1)
    for p in range(1, tail_length + 1):
        if p + 1 <= p + (cycle_length - 1) // 2:
            diff[p + 1] += 2
            diff[(cycle_length - 1) // 2 + p + 1] -= 2
        if cycle_length % 2 == 0:
            diff[cycle_length // 2 + p] += 1
            diff[cycle_length // 2 + p + 1] -= 1
    cur = 0
    for i in range(1, N):
        cur += diff[i]
        ans[i] += cur

def add_between_tails(left_tail: int, right_tail: int):
    # しっぽ↔しっぽ: ショートカットを跨いで距離 s+1
    for s in range(2, left_tail + right_tail + 1):
        lo = max(1, s - right_tail)
        hi = min(left_tail, s - 1)
        ans[s + 1] += max(0, hi - lo + 1)

left_tail_length = X - 1
right_tail_length = N - Y
cycle_length = Y - X + 1

add_line(left_tail_length + 1)               # 左しっぽ＋接続点 X
add_tail_and_cycle(left_tail_length, cycle_length)
add_line(right_tail_length + 1)              # 右しっぽ＋接続点 Y
add_tail_and_cycle(right_tail_length, cycle_length)
add_between_tails(left_tail_length, right_tail_length)
add_cycle(cycle_length)

for i in range(1, N):
    print(ans[i])
```

実装で気をつける境界条件をまとめておく:

| 境界条件 | 該当箇所 | どう扱うか・注意点 |
|---|---|---|
| 接続点 $X$ / $Y$ の二重カウント | `add_line(tail+1)` と `add_tail_and_cycle` の境目 | しっぽに接続点を 1 つ含めて直線として数え、接続点以外のサイクル頂点との組は `add_tail_and_cycle` 側で数えるので重複しない |
| 偶数長サイクルの対角 | `add_cycle` の `2*d == n`、`add_tail_and_cycle` の `cycle_length % 2 == 0` | 真向かいの頂点は 1 つだけ → 半分（`n//2`）／`+1`。off-by-one が出やすいので小ケースで検算したい |
| しっぽが空（$X = 1$ / $Y = N$） | `left_tail` / `right_tail` が 0 | ループ範囲が空、または各項が 0 になり自然に 0。特別扱いは不要 |

計算量: ループは `add_line` が $O(\text{tail})$、`add_cycle` が $O(\text{cycle})$、`add_tail_and_cycle` が $O(\text{tail} + N)$、`add_between_tails` が $O(\text{left} + \text{right})$ で、どれも $O(N)$ 以下。呼び出しは定数回なので全体 **$O(N)$**。想定の $O(N^2)$ と比べて $N$ が大きいほど効く（本問は $N \le 2000$ なので速度上の必要はないが、構造を読む練習として）。

## この問題の面白いところ

- 想定の $O(N^2)$ で**普通に通る**問題なのに、グラフの形をよく見ると $O(N)$ まで落ちる、という**余白**があるのが面白い。
- 「1 ペアずつ距離を測る」発想から、「**しっぽ／サイクルという構造ごとにまとめて数える**」発想への切り替えが気持ちいい。
- 汎用性: 「グラフを直線部分とサイクル部分に分け、カテゴリごとに数え上げる」は、ベースが似た形（パス＋少数の追加辺）の数え上げで効きそう。

## まとめ

- 制約が緩くて $O(N^2)$ で通る問題でも、グラフの形を読むと計算量が一段下がることがある。本問はその 3 部構造（しっぽ＋サイクル＋しっぽ）への分解が決め手だった。
- カギは「距離を 1 ペアずつ測る」のをやめ、**カテゴリごとに距離 $d$ の組数を閉じた式 / 差分配列で数える**こと。
