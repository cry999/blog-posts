## TL;DR

- [ABC126 F - XOR Matching](https://atcoder.jp/contests/abc126/tasks/abc126_f) は、長さ $2^{M+1}$ の「良い数列」を構築する問題
- 想定解法は $x$ と $x \oplus K$ をペアにして並べる構築
- この記事では **`deque` で回文状に組む別解** を紹介する
  - 回文にすると、同じ値の2つの出現の「間」にある要素が左右対称にペアで現れて XOR が打ち消し合い、中心に置いた $K$ だけが残る
  - 末尾だけは $\mathrm{XOR}(0..2^M-1)=0$（$M\ge2$）を使って辻褄を合わせる
- 提出: https://atcoder.jp/contests/abc126/submissions/76901364 （PyPy3, AC）

## 問題概要

- 長さ $2^{M+1}$ の数列 $a$ を作る。$0$ から $2^M-1$ までの各整数が **ちょうど2回ずつ** 出現する
- 「良い数列」の条件: 同じ値が位置 $i, j$（$i<j$）に出現するとき、**その間の要素 $a_{i+1},\dots,a_{j-1}$ の XOR が $K$** になる

:::details 補足: 公式の条件式との対応（両端含む vs 間だけ）
- 公式の条件は両端を含む形 $a_i \oplus a_{i+1} \oplus \cdots \oplus a_j = K$
- ただし $a_i = a_j$ なので両端が打ち消し合い、間だけの XOR $a_{i+1} \oplus \cdots \oplus a_{j-1}$ と一致する
- 本記事では扱いやすい「間の XOR」の形で進める
:::

- 制約: $0 \le M \le 17$、$0 \le K \le 10^9$
- 入出力例:
  - $M=1, K=0$ → `0 0 1 1`（`a_1 xor a_2 = 0`, `a_3 xor a_4 = 0`）
  - 構築不能なら `-1`

最後に、あとで効いてくる前提を一つ押さえておく。$M \ge 2$ のとき $\mathrm{XOR}(0, 1, \dots, 2^M-1) = 0$ が成り立つ。どのビットも $0 \dots 2^M-1$ のうちちょうど半分（$2^{M-1}$ 個）の数で立ち、$M \ge 2$ では $2^{M-1}$ が偶数なので、全ビットが偶数回現れて XOR が $0$ になるからだ。実際 $M=2$ なら $0 \oplus 1 \oplus 2 \oplus 3 = 0$ となる。逆に $M=1$ では $2^{M-1}=1$ が奇数で $0 \oplus 1 = 1 \ne 0$ となり、これがのちの特殊ケースの根っこになる。

## 想定解法（ざっくり）

想定解法は「ペアの作り方」が起点になる。$x$ と $x \oplus K$ をペアにすると、ペア内の2値の XOR は常に $K$ になる（たとえば $M=3, K=5$ なら $(0,5), (1,4), (2,7), (3,6)$）。このペアをうまく並べて条件を満たすのが、想定解法の骨子だ。具体的な構築は [drken さんの解説](https://drken1215.hatenablog.com/entry/2019/05/19/230600) が分かりやすいので、本記事の主題ではないここでは深入りしない。

この記事で扱うのは、これとは別の **「回文で組む」** アプローチである。

## 自作解法: 回文で組む（要点①）

発想の出発点はこうだ。「同じ値の2つの出現の間の XOR を $K$ にしたい」のだから、**間の要素が打ち消し合ってくれれば、中心に $K$ を1個置くだけで済む**。そして打ち消し合いを作る最も素直な形が、**回文（左右対称）** である。

これを踏まえた構築（$M\ge2$ の本体）は次のとおり。

- 中心に $K$ を置く
- $i = 1,2,\dots,2^M-1$（ただし $i=K$ は飛ばす）を、中心の左右に**対称に**1個ずつ積む
  - → `… 3, 2, 1, K, 1, 2, 3 …` のような回文ができる
- 末尾に `0, K, 0` を付け足す（辻褄合わせ用）

末尾の `0, K, 0` は、ここまでで $0$ と $K$ をまだ1回ずつしか置けていないので、$0$ を2回・$K$ の2回目を補うためのものだ。なお $K=0$ のときは $0$ と $K$ が同じ値になって分岐が変わり、末尾は `0` 1個だけになる（コードの `if K != 0`。詳細は証明セクションのアコーディオンを参照）。

```python:submission.py
from collections import deque

M, K = map(int, input().split())

def solve():
    N = 1 << M
    if K >= N:
        return [-1]
    if M == 0:
        return [0, 0]
    if M == 1:
        return [0, 0, 1, 1] if K == 0 else [-1]

    ans = deque([K])
    for i in range(1, N):
        if i == K:
            continue
        ans.append(i)       # 右に積む
        ans.appendleft(i)   # 左に積む（対称）

    if K != 0:
        ans.append(0)
    ans.append(K)
    if K != 0:
        ans.append(0)

    return list(ans)

print(*solve())
```

実装のキモは、`deque` の `append` / `appendleft` で中心から左右に積むだけという点だ。複雑なペアリングは要らない。

## なぜ成り立つか（要点②・証明）

最終形（$K\ne0$ のとき）:

$$
\underbrace{N{-}1,\dots,2,1}_{\text{左}},\ K,\ \underbrace{1,2,\dots,N{-}1}_{\text{右}},\ 0,\ K,\ 0
$$

（$N=2^M$、左右の列からは $K$ を除く）

具体例 $M=2, K=1$ で見ると、数列は `[3, 2, 1, 2, 3, 0, 1, 0]`。値 $3$ のペアの間（`2, 1, 2`）では両端の $2$ が打ち消し合い、中心の $K=1$ だけが残る:

![回文の内側では左右対称な要素がペアで打ち消し合い、中心の K だけが残る様子](/images/20260623-atcoder-abc126-f-xor-matching-palindrome/palindrome-cancel.png)

これを一般化して、3種類の値について間の XOR が $K$ になることを確認する。

- **回文の内側にある値 $i$（$1 \le i \le N-1,\ i \ne K$）**
  - 左ブロックと右ブロックに1個ずつあり、中心 $K$ について左右対称な位置にある
  - 2つの $i$ の間にある要素は、左側の $i-1, i-2, \dots, 1$（$K$ を除く）＋ 中心の $K$ ＋ 右側の $1, 2, \dots, i-1$（$K$ を除く）
  - $1 \le j < i$ を満たす各 $j$（$\ne K$）はちょうど2回現れて打ち消し合い、中心の $K$ だけが1回残る
  - よって間の XOR $= K$ ✓
- **値 $0$**
  - 末尾 `0, K, 0` の2つの $0$。間にあるのは $K$ ただ1つ
  - 間の XOR $= K$ ✓
- **値 $K$**
  - 中心の $K$ と、末尾 `0, K, 0` の $K$
  - 間にある要素は「右の列 $\{1,\dots,N-1\}\setminus\{K\}$」＋「$0$」＝ $\{0,1,\dots,N-1\}\setminus\{K\}$
  - その XOR は $\mathrm{XOR}(0..N-1) \oplus K = 0 \oplus K = K$（$M\ge2$ より）✓
- 出現回数の確認: どの値もちょうど2回 → 数列長 $2N = 2^{M+1}$ も一致 ✓

:::details K=0 のとき（末尾の補正が変わる）
- $K=0$ ではコードの分岐で末尾の `0` 補完が省かれ、最終形は次の形:

$$
\underbrace{N{-}1, \dots, 2, 1}_{\text{左}},\ 0,\ \underbrace{1, 2, \dots, N{-}1}_{\text{右}},\ 0
$$

- 中心の $0$ がそのまま $K$ なので、別途 $K$ を補う必要がない（末尾は $0$ のペアを閉じる $1$ 個だけ）
- **内側の値 $i$**: 間は左右対称、$j<i$ が2回ずつ現れて打ち消し、中心の $0\,(=K)$ が残る → XOR $=0=K$ ✓
- **値 $0$**: 中心と末尾の2つ。間は $1, 2, \dots, N-1$ で、$\mathrm{XOR}(1..N-1) = \mathrm{XOR}(0..N-1) = 0 = K$ ✓
:::

## -1 判定と特殊ケース

- $K \ge 2^M$ → `-1`
  - 数列の要素はすべて $2^M$ 未満（最上位ビットより上は常に $0$）。その部分 XOR も必ず $2^M$ 未満にしかならないので、$K \ge 2^M$ は実現不可能
- $M=0$ → 数列は `0 0` のみ。間に要素なしなので $K=0$ だけ可、それ以外 `-1`
- $M=1$ → どう並べてもペア間 XOR は $0$。$K=0$ なら `0 0 1 1`、それ以外 `-1`
- $M\ge2$ は上の構築で常に作れる（$K<2^M$ なら）

## まとめ

「間の XOR を $K$ にする」という条件は、「**間を打ち消し合わせて、中心に $K$ を1個だけ残す**」と読み替えられる。こう捉え直すと、回文で素直に構築できる。

この解法を支えているのは、XOR の2つの性質だ。

- $a \oplus a = 0$（同じ値を2回 XOR すると消える）→ 左右対称に並べれば、間の要素がペアでまるごと打ち消える
- $\mathrm{XOR}(0, 1, \dots, 2^M-1) = 0$（$M\ge2$）→ 全要素を並べた区間も丸ごと消える

「**打ち消し**」を構築の軸に据えられるのが XOR の面白いところだと思う。どこで何を消し、何を残すかを設計する感覚は、他の XOR 構築問題にも効くはずだ。
