> 自分の理解を再構築するための記事。重みつき Union-Find を初めて学ぶ人にも届けば、くらいの温度感で書く。

## TL;DR

- 普通の Union-Find は「2 つの要素が同じグループか？」しか答えられない。
- **重みつき Union-Find** は、それに加えて「同じグループの 2 要素の**差（相対値）はいくつか**？」にも答えられる。
- 例:「A は B より 3 大きい」「B は C より 2 大きい」から「A は C より 5 大きい」を、**ほぼ定数時間**で判定・計算できる。
- キモは、各ノードに**親との差 `delta`**（親からの相対値）を持たせ、`find` の経路圧縮と `merge` の付け替えのたびに `delta` を正しく付け替えること。

## この記事について

- 対象読者: 普通の Union-Find は知っている／なんとなく使える人。重みつきは「なんか難しそう」で止まっている人。
- ゴール: 各ノードが持つ「`delta`（親との差）」が何を表すのか、`merge` と `find` で `delta` がどう動くのかを分かりやすく図解する。
- この記事で伝えたい 3 点:
  1. 親との差 `delta` とポテンシャルとは何か＝「A は B より 3 大きい」のようなノード間の相対関係を表す
  2. 併合（`merge`）で `delta` をどう計算するか【図解】
  3. 経路圧縮（`find`）で `delta` をどう繰り上げるか【図解】

:::message
このデータ構造は一般に「重みつき Union-Find」と呼ばれるが、本記事では各ノードが持つ値を **`delta`（親との差）** と呼ぶ。`weight`（重み）より「親との差」という意味がはっきりするからだ。
:::

## きっかけ — 辺を足しながら距離を測りたかった

そもそもの動機は「最小全域木を作りながら、同じグループになった 2 ノードの距離をオンラインで知りたい」だった。静的な木なら LCA で距離を測れる（`dist(u, v) = depth(u) + depth(v) − 2·depth(lca)`）が、辺がどんどん増えて（グループが併合されて）いく状況には素直に使いにくい。

そこで「木を構築しながら任意のノード間の距離をはかる方法はないか」と探していて見つけたのが、各ノードに根からの相対値（ポテンシャル）を持たせる重みつき Union-Find だった。「根から測った値の差で 2 ノードの関係を出す」という骨格は、LCA で木の距離を求めるときの考え方とどこか似ている。

## 親との差 `delta` とポテンシャルとは — ノード間の相対関係を表す値【要点③】

重みつき Union-Find が管理するのは、「グループ分け」に加えて「同じグループ内での相対値」だ。そのために、各ノード $x$ に**親ノードとの差** `delta[x]` を持たせる。向きは `delta[x] = potential(x) − potential(親)`、つまり「子 − 親」で、この記事では終始この向きに固定する。

根を基準（ポテンシャル 0）にとると、あるノードの「根からの相対値」は、根までの経路上の `delta` を足し合わせたものになる。これを**ポテンシャル**と呼び、`potential(x)` と書く。イメージは物理の位置エネルギーだ。根を高さ 0 の基準点とし、各ノードは根からの「高さ」を持つ。2 点の相対値はその高さの差で決まる——「ポテンシャル」という呼び名自体、物理に由来する。

同じグループの $x, y$ について求めたいのは、その差 `diff(x, y) = potential(y) − potential(x)`（第 2 引数を第 1 引数から見た相対値）だ。根が同じなら基準（根）の分は打ち消し合うので、2 つのポテンシャルの差だけで相対値が出る。しかも `merge` と向きをそろえてあるので、`merge(x, y, w)` した後に `diff(x, y)` は `w` を返す。設定した関係がそのまま読み出せるわけだ。

この仕組みが使えるのは、2 ノード間の関係が**鎖のように合成できる**ときだ。`A→B` と `B→C` をつなぐと `A→C` が決まり、逆向き `B→A` にもたどれる（可逆）——そんな関係なら扱える。整数の差（`+3` と `+2` をつなぐと `+5`）が代表例で、パリティ（mod 2）や比（「○倍」）なども同じ枠組みに乗る。

## 併合（merge）の仕組み【図解】【要点①】

`merge(x, y, w)` は「$y$ は $x$ より $w$ 大きい」という関係（`potential(y) − potential(x) = w`）を追加する操作だ。手順はこうなる。

1. `find` で $x, y$ それぞれの根 $rx, ry$ と、`potential(x)`, `potential(y)` を求める。
2. すでに同じグループ（$rx = ry$）なら、そのまま終わり。矛盾判定（`potential(y) − potential(x)` が $w$ と一致するか）が要る問題ではここでチェックするが、その扱いは問題ごとに異なるので本記事では省く。
3. 違うグループなら、片方の根をもう片方の根の子にする。このとき**新しい辺の `delta`** を、関係式が保たれるように計算する。

新しい辺の `delta` は次のように導ける（$ry$ を $rx$ の下にぶら下げる場合）。

1. **保ちたいこと**: 併合したあとも、追加する関係 `potential(y) − potential(x) = w` が成り立ってほしい。
2. **ぶら下げると 0 の位置が 1 つになる**: ポテンシャルは「根を 0 とした値」だ。併合前は `rx` と `ry` に 0 の位置が 2 つあるが、`ry` を `rx` の子にすると 0 の位置は `rx` だけになり、全員が `rx` から測り直される。`ry` は根でなくなり `rx` の子になるので、これまで自分が 0 の位置だった `ry` のポテンシャルは `rx` から見た値になる（`0 → delta[ry]`）。この `delta[ry]` が、新しく張る辺の値だ。
3. **各ノードの新しいポテンシャル**（`rx` を 0 として測り直す。下図参照）:
   - `x` は `rx` 側なので変わらない → `potential(x)`
   - `ry` の新しいポテンシャル → `delta[ry]`
   - `y` は `ry` にぶら下がったまま。`y` と `ry` の差は併合で変わらないので、`y` の新しいポテンシャル → `potential(y) + delta[ry]`（元の値に、0 の位置が `ry`→`rx` に移るぶん `delta[ry]` を足す）

![各ノードの新しいポテンシャル: 辺は親との差 delta、ノードの値は根 rx から delta を足し上げた potential。y は delta[ry] + delta[y] の 2 本ぶん](/images/20260702-weighted-union-find/merge-potentials.png)

あとは 1. の式に 3. を代入すると

$$
(potential(y) + delta[ry]) - potential(x) = w
$$

となり、`delta[ry]` について解くと

$$
delta[ry] = w + potential(x) - potential(y)
$$

が得られる。

:::message
サイズ併合で「どちらをどちらにぶら下げるか」を選ぶ場合、ぶら下げる向きが逆になると `delta` の符号も反転する。ぶら下げる子が `rx` 側になるので、`delta` は上で求めた値の符号を反転したものになる。
:::

## 経路圧縮（find）の仕組み【図解】【要点②】

普通の `find` は「根を返しつつ、通ったノードを根に直結」させる。重みつきでは、`delta` は常に「親との差」なので、経路圧縮で親が根に変わると、新しい `delta` は根との差——つまり根からそのノードまでの `delta` の累積（ポテンシャル）——に書き換わる。

再帰版のイメージはこうだ。

1. `find(親)` を呼び、**根から親までの `delta` の累積（＝親のポテンシャル）** を得る。
2. 自分の `delta[x]`（親との差）に親のポテンシャルを足すと、**根から自分までの `delta` の累積（＝自分のポテンシャル）** になる。
3. `parent[x]` を根に付け替え、`delta[x]` をこのポテンシャルで上書きする。

ポイントは、付け替え前は「親との差」、付け替え後は「根との差」で、基準が親から根に変わることだ。だから通ってきた `delta` を足し込んで更新する。これを怠ると、木の形が変わったのに `delta` が古い親基準のままになり、`potential` が壊れてしまう。

![find の経路圧縮をステップで追う GIF: 鎖 R←P←X を根まで辿って delta を累積し、X を根 R に直結して delta を「根からの累積」に付け替える 5 ステップ](/images/20260702-weighted-union-find/path-compression-steps.gif)

## 実装（Python）

配列は 2 本だ。`parent`（親。根なら自分自身）と `delta`（親との差）を持つ。`find` は経路圧縮しながらポテンシャルを更新し、`diff(x, y)` は同じグループ内で `potential(y) − potential(x)` を返す。

```python
class WeightedUnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.size = [1] * n   # 各グループの要素数（サイズ併合用）
        self.delta = [0] * n  # delta[x] = potential(x) - potential(parent[x])（親との差）

    def find(self, x):
        # 経路圧縮しながら、根とポテンシャルを更新する
        if self.parent[x] == x:
            return x
        root = self.find(self.parent[x])
        self.delta[x] += self.delta[self.parent[x]]  # 根からの累積に付け替え
        self.parent[x] = root
        return root

    def potential(self, x):
        self.find(x)          # 経路圧縮して delta[x] を根基準に
        return self.delta[x]

    def diff(self, x, y):
        # potential(y) - potential(x)（同じグループ前提）
        return self.potential(y) - self.potential(x)

    def merge(self, x, y, w):
        # potential(y) - potential(x) = w となるよう併合
        w += self.potential(x) - self.potential(y)
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return  # すでに同じグループ。矛盾判定が要る問題ではここで行う（扱いは問題次第なので本記事では省く）
        if self.size[rx] < self.size[ry]:
            rx, ry, w = ry, rx, -w  # 小さい木を大きい木の下に。向きが変わると符号反転
        self.parent[ry] = rx
        self.delta[ry] = w
        self.size[rx] += self.size[ry]
```

:::message
実装で気をつける点:

- 差の向きを最初に固定しないと符号地獄になる。本記事では `delta` ＝ 子 − 親、`diff(x, y) = potential(y) − potential(x)` に固定した。
- `merge` で向きを入れ替えたら `delta` も符号反転を忘れない。
- `potential(x)` を呼ぶ前に `find(x)` で経路圧縮しておく（コード上は `potential` 内で呼んでいる）。
:::

## 使いどころ

重みつき Union-Find が刺さるのは、たとえば次のような場面だ。

- 「$x_i$ と $x_j$ の差がちょうど $d$」という制約が多数与えられ、任意ペアの差を答えたい問題。
- パリティ（mod 2）の管理: 「$x$ と $y$ は異なる／同じ」を `delta` mod 2 で表現する（2 部グラフ判定など）。
- 累積的な相対関係を効率よく畳み込みたい場面全般。

## まとめ

重みつき Union-Find は、普通の Union-Find に「ノード間の相対値」を足したものだ。各ノードに親との差 `delta` を持たせ、`find`（経路圧縮）と `merge`（併合）のたびに `delta` を正しく更新する——本質はそれに尽きる。図で「基準が親から根に変わる」感覚さえ掴めれば、符号や更新式は自然に導けるはずだ。
